# Bolt Journal
## 2023-11-20 - [Fixing N+1 Queries on Items Batch Operations]
**Learning:** Found multiple instances where the application iterates sequentially using a `for` loop to execute database queries. Specifically, `src/modules/item/item.model.ts` had a `for` loop that ran sequential validation and insertions when creating/updating items.
**Action:** Replace looped individual db calls in Drizzle with batch reads using `where(inArray(table.id, arrayIds))` + a `Map` structure to enable O(1) in-memory checks, and bulk insert values array to cut N queries down to a single query. Check codebase for loops when doing any CRUD operation.

## 2026-07-18 - [Fixing N+1 Queries on Transaction Stock Updates]
**Learning:** The `transaction.service.ts` file iterates sequentially using a `for` loop to execute database queries for updating `inventoryStocks` when completing or reverting transactions. Additionally, optimizing with `Promise.all()` for concurrently executing database update operations requires carefully maintaining an in-memory aggregation map, as subsequent operations in the loop could otherwise override quantity updates if multiple transaction line items belong to the same parent item.
**Action:** Replaced sequential `SELECT` DB queries inside loops with batch reads using `inArray()`. Processed all line items by managing an accurate in-memory quantity map and queued all DB insertions/updates into an array, executing them simultaneously using `Promise.all()`. When utilizing `Promise.all()`, ensure duplicate items in the loop are fully accounted for via an isolated memory tracker so updates to the same entity stack correctly.
