# issues-data-order-from-tiktok

Buatkan issues-data-order-from-tiktok-shop.md di folder issues yang berisi perencanaan untuk nanti di implementasikan oleh junior programmer atau AI model yang lebih murah.

Isi dari planningnya adalah sebagai berikut :

Buatkan planning untuk membuat background job / console app, yang melisten webhook dari tiktok shop atau e-commerce lainnya yang saya punya untuk menarik semua data pesanan yang pending ke aplikasi Warehouse Management System saya. agar nantinya bisa di validasi oleh orang packing atau gudang untuk mencocokan datanya sudah di packing atau belum. Flow background job nya itu sebagai berikut :
1. Menarik pesanan yang statusnya pending / belum di bayar. dari tiktok shop
2. Setelah di tarik ke aplikasi saya, saya ingin nanti ada kolom status pending, dalam proses, selesai.
3. saya akan buat menu / halaman untuk bisa upload resi / bukti pengiriman untuk pesanan yang status pending / dalam proses.
4. dan ketika pesanan di update statusnya menjadi selesai, maka pesanan otomatis akan terupdate juga di tiktok shop.
5. untuk akun tiktok shop ada 3 akun, jadi saya ingin bisa membedakan pesanan dari akun tiktok shop mana yang masuk.

buatkan table order_platform, dengan kolom sebagai berikut :
1. id
2. platform_id
3. status
4. total_amount
5. shipping_provider
6. tracking_number
7. buyer_message
8. shipping_type
9. rts_sla_time
10. is_cod
11. cancel_reason
12. create_time
13. update_time

buatkan table order_platform_detail, dengan kolom sebagai berikut :
1. id
2. order_platform_id
3. product_id
4. product_name
5. seller_sku
6. sale_price
7. qty
8. sku_id

buatkan table recipient_platform, dengan kolom sebagai berikut :
1. id
2. order_platform_id
3. name
4. phone
5. address
6. region_code
7. postal_code 

pastikan untuk data yang di tarik dan di store ke database di validasi terlebih dahulu agar tidak ada duplicate data dan tidak ada data yang double, dan jika ada data yang tidak valid agar bisa di handle dengan baik, bisa di skip atau di queue terlebih dahulu untuk di validasi ulang.

buatkan mapper untuk masing - masing platform e-commerce, dengan mengkonversi semua data yang masuk ke format data saya. pastikan tidak ada yang miss dan berikan nilai default jika tidak ada key atau valuenya agar tidak bermasalah ketika di insert ke database.

Tolong buatkan Workflow terlebih dahulu agar saya bisa review dan approve untuk implementasi planning di atas.
